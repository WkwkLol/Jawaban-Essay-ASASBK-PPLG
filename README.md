## Nomor 4

<p>
- Waterfall <br>
  Metode ini fokus sesuai urutan seperti air terjun. Tahap sebelumnya harus selesai sebelum lanjut ke tahap selanjutnya. <br>
  Bagus untuk project yang sudah di rancang dengan jelas. <br>
- Agile <br>
  Metode ini cocok jika project banyak perubahan dan penyesuaian yang cepat. <br>
- V-Model <br>
  Ada sisi kiri "verifikasi" dan kanan "validasi", mereka ber 2 terhubung dalam tahap code atau implementasi. <br>
  Cocok untuk project kompleks, ketat, dan berkualitas tinggi dikarenakan ada pengujian di tiap tahapnya.
</p>
